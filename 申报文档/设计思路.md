# 设计思路

## 一、总体设计理念

### 核心理念：游戏化学习（Gamification）

传统的教学工具往往是被动的——看视频、读文章、做题库。本作品采用**"玩家即开发者"**的游戏化设计：

> 你不是在观看别人训练AI，你就是在亲自训练AI。

每个关卡的设计都遵循 **"做决策 → 看反馈 → 理解原理"** 的循环：

```
┌──────────┐    ┌──────────┐    ┌──────────┐
│  做出决策  │ → │  观察反馈  │ → │  理解原理  │
│ (选择/判断) │    │ (可视化结果)│    │ (知识卡片)│
└──────────┘    └──────────┘    └──────────┘
       ↑                                  │
       └──────────── 进入下一关 ────────────┘
```

### 设计原则

1. **无前置知识要求**：不需要编程或医学背景
2. **即时反馈**：每次操作立刻看到可视化结果
3. **容错设计**：错误的判断不会扣分，反而提供学习机会
4. **渐进式复杂度**：关卡1最简单（纯粹的观察和调节），关卡5最综合（独立诊断 + 识别AI陷阱）

## 二、关卡设计逻辑

### 关卡1：数据猎人 — 认知阶段

**设计意图**：让用户先"认识数据"，消除对CT图像的陌生感。

- Tab1（认识CT数据）：知识卡片 + 3D切片浏览器，理解"CT是什么"
- Tab2（肺窗调节实验）：可调滑块 + 实时预览，理解窗技术的意义
- Tab3（训练数据探秘）：浏览专家标注好的样本，理解"标注数据"的概念

**设计亮点**：不要求用户具备医学知识去"判断结节"，而是以"观察者"的身份浏览已标注的数据。降低门槛的同时传递"标注数据是AI训练的基础"这一核心概念。

### 关卡2：架构师 — 选择阶段

**设计意图**：把抽象的"模型选型"变成可视化的架构蓝图对比。

- 两种架构卡片对比：ResNet18 vs 3D CNN，各自显示优缺点
- 超参数滑块：学习率、批次大小、训练轮数，每个旁边有通俗解释
- 实时参数统计：选择不同配置后，参数量/训练时间/显存需求动态更新

**设计亮点**：用户不需要理解卷积的数学公式，只需要理解"不同的架构适合不同的任务"这个概念。

### 关卡3：训练大师 — 核心体验阶段 ⭐

**设计意图**：这是整个作品的**核心体验**——可视化AI的学习过程。

- **截断式曲线图**：只显示到当前轮次的loss/acc曲线，模拟"正在进行"的感觉
- **三个决策点**：在训练的关键时刻，让用户做出选择（保持学习率/调整/早停）
- **Grad-CAM演变**：每个epoch的热力图从"弥散→聚焦"，直观展示AI注意力的变化
- **情景化说明**：每个阶段有独立的文字解释当前发生了什么

**设计亮点**：
1. 曲线只显示"已发生"的部分（如第5轮时只显示1-5轮的曲线），营造实时感
2. 决策有后果——不同选择导致不同最终准确率
3. Grad-CAM演变是整个作品最创新的可视化：从epoch 1的"6-8个随机热点"到epoch 50的"精准聚焦结节"，一眼就能看懂

### 关卡4：诊断专家 — 应用阶段

**设计意图**：将前面学到的知识应用到实际诊断中。

- 独立阅片 + AI分析 + 人机对比三步走
- IoU评分量化"你的判断"和"AI的判断"的重合度
- 每个病例附带医学知识点（影像学特征、鉴别诊断）

**设计亮点**：用户先自己判断，再看AI的判断，形成对比。既学到了医学知识，又理解了AI辅助诊断的场景。

### 关卡5：临床实战 — 综合挑战阶段

**设计意图**：在模拟临床环境中检验所有学到的知识。

- 批量阅片界面（12张CT）
- AI辅助可开关（模拟真实场景中AI并非100%可用）
- **AI陷阱**：故意设置2个AI误判的病例，测试用户是否能独立判断

**设计亮点**：AI陷阱是本关的点睛之笔——如果用户盲从AI，结算面板会指出"AI不是万能的"，传递最核心的科普信息。

## 三、交互设计

### 侧边栏状态系统

贯穿全部关卡的状态面板：
- 进度条（5关）
- 实时分数
- 模型准确率
- 成就系统（9个成就）
- 开发者模式（可跳关，方便演示）

### 反馈机制

- **正向反馈**：正确操作 → 加分 + 气球动画 + 成就解锁
- **负向反馈**：错误判断 → 不扣分 + 解释为什么错 + 学习提示
- **进度可视化**：每关有进度追踪器，完成的部分打勾

## 四、技术选型

| 技术 | 选择理由 |
|------|---------|
| **Streamlit** | Python原生的Web框架，无需前后端分离，适合数据科学/ML项目快速构建交互式界面 |
| **PyTorch** | 最流行的深度学习框架，ResNet18预训练模型开箱即用 |
| **Matplotlib** | 训练曲线可视化 |
| **OpenCV** | Grad-CAM热力图生成和图像处理 |
| **Streamlit Cloud** | 免费在线部署，自动从GitHub同步 |

## 五、视觉设计

- **深色科技风主题**：深蓝背景 + 青色强调色，营造"AI实验室"氛围
- **等宽字体的数据面板**：模拟终端/开发环境的感觉
- **渐变色热力图**：Jet colormap 从蓝(冷)到红(热)，直观展示注意力强度
